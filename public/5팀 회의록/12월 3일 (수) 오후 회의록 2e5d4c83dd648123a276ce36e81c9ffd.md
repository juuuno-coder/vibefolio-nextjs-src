# 12월 3일 (수) 오후 회의록

### 회의 개요

- 참가자들이 코드 및 창작물 공유 플랫폼 개발을 위한 참고 사이트들을 검토하고 발표
- 주요 목표는 비기너 개발자들을 위한 커뮤니티 형성 및 코드 공유 플랫폼 구축
- 각 참가자가 조사한 참고 사이트의 특징과 기능을 공유

### CodePen 분석

- 코드 공유 및 판매 플랫폼으로, 개발 중인 서비스와 가장 유사한 사이트
- 사용자들이 작업물을 공유하고 코드와 실시간 효과를 동시에 확인 가능
- 직접 코드 편집 기능을 제공하여 즉각적인 결과 확인 가능
- 무료/유료 구독 모델을 통한 수익화 구조
- 프로필 팔로우 기능을 통한 커뮤니티 형성 지원

### BNC(Behance) 분석

- 어도비에서 개발한 세계 최대의 온라인 창작 포트폴리오 공유 플랫폼
- 디자인, 일러스트, 사진, UX, UI, 모션 그래픽 등 시각적 작업 위주의 포트폴리오 공유
- 프로젝트 업로드 개수에 제한 없음
- 유료 구독자에게는 더 넓은 도달 범위 제공

### 주요 기능 요구사항

- 직관적인 UI로 사용자 경험 개선 필요
- 코드 입력 시 실시간으로 효과를 확인할 수 있는 기능
- 커뮤니티 형성을 위한 좋아요, 댓글, 조회수 등의 상호작용 기능
- 사용자 간 프로젝트 공유 및 챌린지를 통한 실력 향상 기회 제공
- 모바일 환경에서의 화면 디스플레이 지원

### 비즈니스 모델 고려사항

- 초기 단계에서는 사용자 유치가 수익화보다 중요
- 다양한 수익화 방식 검토 (마일리지/패킷 판매, 구독 모델 등)
- 무료 사용자와 유료(프로) 사용자 간 접근 권한 차등화
- 비기너 개발자 대상 커뮤니티 형성을 통한 지속적인 참여 유도

### 참고 플랫폼 분석

- 드리블(Dribbble): 디자이너들의 인스타그램 같은 느낌으로, 작품이 나열되어 있고 작가 페이지에서 포트폴리오와 디자인 작품을 볼 수 있으며 리액션 기능 제공
- 프레이머(Framer): 디자인부터 사이트 제작까지 가능한 플랫폼으로, UI가 직관적이고 반응형이 좋으며 다양한 컴포넌트 제공
- 여러 포트폴리오 템플릿 및 웹페이지 제작 도구 검토
- AI 기능을 활용한 페이지 구성 및 세밀한 정렬 기능 등 최신 기술 적용 사례

### 타겟 사용자 및 방향성

- 풀스택 디자이너를 주요 타겟으로 설정
- 비주얼라이징 능력이 있는 사용자들을 위한 BNC 스타일의 UI 구현
- 사용자 간 교류를 통해 부족한 부분을 채우고 서로의 능력을 나눌 수 있는 상호작용 가능한 플랫폼 지향

### UI/UX 설계 원칙

- 직관적인 시각 요소를 통해 프로젝트 특성과 기능을 명확히 전달해야 함
- 텍스트 위주의 단조로운 구성보다 비주얼적으로 흥미를 유발하는 방식 필요
- 아이콘, 닉네임, 태그만으로는 사용자의 호기심과 참여를 유도하기 어려움
- 과도하게 많은 정보 요구와 직관적이지 않은 UI는 사용자 이탈의 원인

### 개선 방향

- 흥미로운 아이디어를 비주얼라이징된 UI로 제공하여 재방문 유도
- 사용자 간 협업을 촉진하는 인터페이스 설계 (예: 도움이 필요한 부분을 표시하여 다른 사용자의 참여 유도)
- 목적과 기능이 명확하게 구분되는 UI 설계로 사용자 혼란 방지
- 비엔스(BNC) 스타일의 UI를 활용한 프로젝트 매력 증대

### 참고 사이트 분석

- 아이디어스(Idus): 작가들이 다양한 상품을 등록하고 판매하는 플랫폼 검토
- 아이디어스는 작가 신청 후 상품성이 있다고 판단되면 등록 허가
- 상품 등록 시 사진, 작품명, 가격, 할인가, 수량 등 정보 입력
- OKKY: 개발자 커뮤니티 사이트 - Q&A, 지식, 테크 뉴스 등의 메뉴 구성
- 트래픽 증가를 위해 테크뉴스나 IT 보도자료 같은 콘텐츠 추가 고려

- AI 창작자/바이브 코더들을 위한 포트폴리오/창작물 공유 플랫폼으로 방향 전환 제안
- 노트폴리오를 베이스로 하되, AI 창작자를 타겟으로 차별화

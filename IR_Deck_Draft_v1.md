## 슬라이드 2: 문제 제기 (The Problem)

**레이아웃:**

- 분할 화면 (왼쪽: 텍스트, 오른쪽: 일러스트/아이콘)

**내용:**

- **헤드라인:** 폭발하는 AI 창작물, 갈 곳 잃은 데이터
- **핵심 문제점:**
  1.  **정보의 파편화 (Scattered):** 수많은 공모전과 채용 정보가 검증되지 않은 채 여러 사이트에 흩어져 있습니다.
  2.  **신뢰도 하락 (Broken Trust):** 저화질 이미지, 깨진 링크, 부정확한 정보로 사용자 경험이 저해됩니다.
  3.  **낙후된 UX:** AI 아트의 아름다움을 담아내지 못하는 구시대적 인터페이스가 창작 의욕을 꺾습니다.

**시각 요소:**

- "혼란", "깨진 링크", "구형 UI"를 상징하는 아이콘
- 복잡하고 정리되지 않은 타사 경쟁 사이트 예시 화면 (X 표시)

## 슬라이드 3: 솔루션 (Our Solution)

**레이아웃:**

- 3단 카드 레이아웃

**내용:**

- **헤드라인:** 바이브폴리오: 프리미엄 큐레이션 플랫폼
- **Card 1: 심층 큐레이션 (Deep Curation)**
  - "단순 나열이 아닙니다. 심층 크롤링으로 공식 링크, 상금, 응모 대상까지 완벽하게 검증합니다."
- **Card 2: 자산 아카이빙 (Asset Archiving)**
  - "사라지지 않는 데이터. Supabase 스토리지를 통해 모든 포스터와 이미지를 안전하게 자동 백업합니다."
- **Card 3: AI-First UX**
  - "AI 아트워크에 최적화된 글래스모피즘 디자인과 전용 뷰어로 압도적인 몰입감을 제공합니다."

**시각 요소:**

- 바이브폴리오의 '공모전 상세 페이지' 스크린샷 (깔끔한 UI 강조)
- 아이콘: 데이터베이스 + 방패(보안), 마법봉(큐레이션)

## 슬라이드 4: 핵심 기술 (Core Technology)

**레이아웃:**

- 다이어그램 / 플로우차트 스타일

**내용:**

- **헤드라인:** '정공법(Standard Approach)'으로 설계된 기술 아키텍처
- **핵심 엔진:**
  - **1단계: 지능형 크롤링:** Cheerio 파서가 상세 페이지까지 방문하여 숨겨진 정보를 찾아냅니다.
  - **2단계: 오토 필(Auto-fill):** 링크 하나만으로 메타 데이터를 분석해 제목, 마감일, 상금을 자동 입력합니다.
  - **3단계: 보안 아카이빙:** 이미지를 프록시 서버를 통해 안전하게 저장하며(RLS), 외부 링크 소실에 대비합니다.
- **기술 스택:** Next.js 14 (App Router), Supabase (PostgreSQL), TypeScript

**시각 요소:**

- 시스템 아키텍처 다이어그램: 원본 사이트 -> 지능형 크롤러 -> Supabase DB -> 바이브폴리오 UI
- 기술 로고: Next.js, Supabase, TypeScript

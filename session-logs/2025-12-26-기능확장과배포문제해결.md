[제목]: 흐려진 경계와 선명한 구조
(부제: 2025-12-26, 레이아웃 개선 및 기능 모듈 통합)

발단: 겹쳐진 레이어
화면 상단은 혼잡했다. `TopHeader`와 `Header`의 경계가 모호하여 스크롤 할 때마다 콘텐츠가 가려지거나 겹치는 현상이 발생했다. 기능적으로도 미완성이었다. 프로젝트 상세 모달의 '제안하기' 버튼은 맥락에 맞지 않았고, 마이페이지에는 프로필 관리와 문의 내역을 확인할 공간이 부재했다. 레오는 사이트의 뼈대가 흔들리고 있다고 느꼈다.

전개: 수직의 질서
안티그래비티는 `z-index`와 `height` 값을 재조정하여 헤더의 위계질서를 바로잡았다. `TopHeader`는 더 얇고 날렵하게 최상단에 고정시켰고, 그 아래 `Header`와 `StickyMenu`가 계산된 높이만큼 자리를 잡도록 했다. 레이아웃이 안정되자 곧바로 기능 확장에 착수했다. 모달의 버튼을 '1:1 문의하기'로 변경하고, 이를 받아줄 `mypage/inquiries`와 `mypage/profile` 페이지를 신설하여 흩어진 기능들을 마이페이지라는 하나의 지붕 아래로 모았다.

위기: 접근의 문턱
기능은 만들어졌으나 접근 제어가 허술했다. 로그인하지 않은 사용자가 업로드 버튼을 누르거나 프로필 페이지에 진입할 수 있었다. 또한, 채용 정보와 공모전 목록이 뒤섞여 있어 정보를 찾기 어려웠다. 이는 단순한 불편함을 넘어 데이터 무결성을 해칠 수 있는 보안 이슈였다.

절정: 명확한 구분
안티그래비티는 접근 제어 로직을 강화했다. 업로드 페이지 진입 시 로그인 여부와 프로필 존재 유무를 확인하여, 자격이 없는 접근은 즉시 로그인 페이지로 리다이렉트 시켰다. `src/app/recruit/page.tsx`는 탭 컴포넌트를 도입하여 채용(Job), 공모전(Contest), 이벤트(Event)를 명확히 분리했다. 이제 사용자는 자신이 원하는 정보를 헤매지 않고 찾을 수 있게 되었다.

결말: 단단해진 기둥
헤더는 깔끔하게 정리되었고, 마이페이지는 사용자의 활동을 담아내는 그릇이 되었다. 관리자 대시보드 또한 통계와 관리 기능을 갖추며 사이트 운영의 관제탑 역할을 하게 되었다. 레오는 "이제야 시스템이 유기적으로 돌아가는 것 같다"며 안도했다. 흐릿했던 기능의 경계들이 선명한 구조로 다시 태어난 순간이었다.

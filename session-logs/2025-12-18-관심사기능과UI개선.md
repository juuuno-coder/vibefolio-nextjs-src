[제목]: 관심사의 발견과 초록빛 테마
(부제: 2025-12-18, UI 테마 개선 및 관심사 필터링 구현)

발단: 색바랜 정체성
프로젝트의 전반적인 톤앤매너가 흔들리고 있었다. 곳곳에 하드코딩된 민트색과 인디고색 코드는 브랜드의 통일성을 해치고 있었고, 헤더는 너무 비대하여 콘텐츠를 가리기 일쑤였다. 무엇보다 레오는 사용자가 자신의 취향에 맞는 프로젝트를 찾을 수 있는 '관심사(Interests)' 기능이 껍데기만 있는 상태라고 지적했다.

전개: 다이어트와 염색
안티그래비티는 먼저 헤더의 높이를 `h-20`에서 `h-16`으로 줄여 화면의 숨통을 틔웠다. `StickyMenu` 또한 스크롤 시 상단에 착 달라붙도록 위치를 미세 조정했다. 그리고 코드 전반에 흩어져 있던 색상 값들을 Tailwind CSS의 유틸리티 클래스와 새로운 'Green' 테마로 치환했다. 이제 사이트는 일관된 초록빛 정체성을 띠기 시작했다.

위기: 정의되지 않은 변수
관심사 필터를 클릭했을 때 사용자를 안내할 모달을 구현하던 중, `ReferenceError: interestModalOpen is not defined`라는 거친 경고가 떴다. 급하게 기능을 추가하다가 상태 변수 `useState` 선언 위치를 잘못 잡은 탓이었다. JS 엔진이 변수를 찾지 못해 헤매고 있었다. 또한 Vercel 배포 과정에서 TypeScript는 타입이 명확하지 않은 변수들(Implicit Any)을 용납하지 않았다.

절정: 명확한 안내
안티그래비티는 `useState`를 컴포넌트 최상단으로 끌어올려 스코프 문제를 해결했다. 관심사가 설정되지 않은 사용자에게는 'Shadcn UI Dialog'를 활용한 깔끔한 모달을 띄워 마이페이지로 자연스럽게 유도했다. 빌드 에러를 잡기 위해 `AuthChangeEvent`와 `Session` 타입을 명시적으로 선언하여 TypeScript의 엄격한 기준을 통과했다.

결말: 취향의 길잡이
이제 사용자는 깔끔해진 헤더 아래에서 자신의 관심사에 맞는 프로젝트를 필터링할 수 있게 되었다. 설정이 안 되어 있다면 친절한 모달이 길을 안내했다. 레오는 시각적으로 정돈되고 기능적으로 친절해진 사이트를 보며, UI/UX 개선이 기술적 안정성만큼이나 중요하다는 것을 실감했다.

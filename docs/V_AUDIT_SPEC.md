# 🚀 V-Audit: 범용 피드백 솔루션 기능 개발 명세서

## 1. 개요 (Vision)

기존의 '포트폴리오 리뷰'라는 좁은 틀을 벗어나, 창작자나 기획자가 **사진, 영상, 링크 등 어떤 형태의 결과물**이든 스스로 피드백 받고 싶은 항목을 설정하여 원하는 타겟에게 배포하고, 이를 흥미로운 데이터(레이더 차트, 투표 등)로 시각화하여 즐기는 **'피드백 플레이그라운드'**로 진화합니다.

---

## 2. 핵심 기능 정의

### 2.1. 유연한 콘텐츠 업로드 (Flexible Media Support)

- **지원 제형**: 외부 URL(웹사이트), 고화질 이미지(단일/슬라이드), 영상(YouTube/Vimeo/Direct), 디자인 시안 등.
- **기능**: 업로드 시점에 프리뷰 형식을 선택하여 뷰어에서 최적의 해상도와 비율로 노출.

### 2.2. 사용자 정의 평가 항목 (Customizable Audit Items)

- **현재**: 고정된 4대 항목 (기획력, 완성도, 독창성, 상업성)
- **변경**: 프로젝트 생성자가 **최대 6개까지 본인이 직접 평가 받고 싶은 항목**을 정의.
  - 예: "색감", "폰트 가독성", "BGM 조화", "사용자 편의성" 등으로 자유롭게 설정 가능.

### 2.3. 개인화된 배포 및 타겟팅 (Smart Distribution)

- **Unlisted/Private 배포**: 전체 공개가 아닌 전용 링크(Secret URL)를 통해 특정 그룹에게만 피드백 요청.
- **QR 코드 생성**: 오프라인이나 SNS에서 즉시 참여 가능한 QR 코드 및 공유 카드 생성.

### 2.4. 데이터 아카이빙 및 인사이트 (Statistics & Fun)

- **실시간 레이더 차트**: 피드백이 쌓일 때마다 변화하는 '성장형 차트' 제공.
- **AI 레포트 브리핑**: 수집된 정량 데이터를 바탕으로 AI가 핵심 장점과 보완점을 한 줄로 브리핑.

---

## 3. 기술적 구현 계획

### 3.1. 데이터베이스 구조 고도화 (`Project` 테이블 활용)

- `custom_data` JSON 필드를 적극 활용하여 스키마 변경 없이 유동성 확보:
  ```json
  {
    "audit_type": "image_slide" | "video" | "link",
    "custom_categories": [
      { "id": "score_1", "label": "색감", "icon": "Palette" },
      { "id": "score_2", "label": "가독성", "icon": "Type" }
    ],
    "review_steps": ["rating", "voting", "proposal"],
    "poll_options": ["기대돼요", "아직은 좀..", "더 공부하세요"]
  }
  ```

### 3.2. 프론트엔드 작업 리스트

1.  **동적 평가 컴포넌트 (`MichelinRating`)**: `custom_categories` 데이터에 따라 레이더 차트 꼭짓점과 슬라이더 자동 생성.
2.  **콘텐츠 프리뷰 엔진 (`ViewerArea`)**: 미디어 타입에 따라 `iframe`, `ImageSlider`, `VideoPlayer`를 조건부 렌더링.
3.  **간편 설정 UI**: 기존 업로드 페이지에 '진단 항목 설정' 단계 추가 (또는 전용 진단 생성기 개발).

---

## 4. 유기적 연결성 (Integration)

- 사용자가 프로젝트를 올릴 때 "이거 피드백 좀 받아보고 싶은데?" 버튼 클릭 한 번으로 V-Audit 설정으로 바로 진입.
- 수집된 피드백은 '나의 아카이브' 탭에서 시계열 통계로 확인 가능.

---

## 5. 단계별 실행 계획

1.  **Step 1**: `MichelinRating`과 `FeedbackPoll`을 데이터 기반의 동적 컴포넌트로 전환 (현재 작업 예정).
2.  **Step 2**: 이미지/영상 프리뷰 기능 보강.
3.  **Step 3**: '진단 항목 커스텀 설정' 페이지 및 아카이브 대시보드 구축.

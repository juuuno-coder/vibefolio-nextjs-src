## 🔍 상세 분석

이번 최적화 작업은 `src/app/project/upload/page.tsx`에서 초기 로딩 시 즉시 필요하지 않은 컴포넌트들을 **Code Splitting (코드 분할)** 하는 데 중점을 두었습니다.

### 1. 주요 변경 사항

- **Dynamic Imports 적용:** 기존에 상단에서 정적(`import`)으로 불러오던 무거운 모달 및 관리 컴포넌트들을 `next/dynamic`을 사용하여 동적(`import()`)으로 변경했습니다.
  - `EmbedModal`, `AssetModal`, `StyleModal` (에디터 내 기능 모달)
  - `PhotoGridModal`, `LightroomModal` (이미지 관련 대형 모달)
  - `CollaboratorManager` (공동 제작자 관리)
  - `LeanCanvasModal` (AI 기획 도구)

### 2. 성능 개선 효과

- **초기 로딩 속도 (TTI) 향상:**
  - 사용자가 페이지에 진입하자마자 다운로드해야 하는 자바스크립트 파일의 크기가 **233kB에서 196kB로 약 16% 줄어들었습니다.**
  - 이는 특히 모바일 네트워크 환경이나 저사양 기기에서 페이지가 인터랙티브해지는 시간(Time to Interactive)을 단축시키는 데 직접적인 기여를 합니다.
- **리소스 효율성 증대:**
  - 사용자가 "공동 제작자 추가" 버튼이나 "포토 그리드" 기능을 클릭하기 전까지는 관련 코드를 다운로드하지 않습니다.
  - 불필요한 데이터 전송을 막아 대역폭을 절약합니다.

### 3. 검증 방법

- 동일한 환경에서 소스 코드 변경 전/후의 `npm run build` 결과를 비교 분석하였습니다.
- Next.js의 빌드 아웃풋(First Load JS)을 기준으로 측정되었습니다.

# 프로젝트 에디터 고도화 및 개선 계획

## 1. 현상 진단 및 문제점 분석

### 1.1 "커서가 누워있는" 현상 (시각적 불편)

- **원인**: 현재 적용된 CSS 및 `Tailwind Typography` 플러그인의 기본 스타일이 인용구(`blockquote`)나 특정 텍스트 블록에 `font-style: italic`을 강제하고 있어, 커서(`caret`) 또한 기울어져 보이는 현상입니다.
- **진단**: 기능적 오류는 아니나, 사용자 입장에서 "입력 모드가 잘못되었다"고 느끼게 하여 집필 경험을 저해합니다.

### 1.2 UX/UI 사용성 한계

- **블록 개념 부재**: 최신 에디터(Notion, Medium)는 '블록(Block)' 단위 편집을 지원하여 문단 순서 변경이 쉽지만, 현재는 전통적인 '문서형(What You See Is What You Get)' 방식이라 구조 변경이 어렵습니다.
- **마우스 의존도**: 제목, 리스트, 이미지 등을 넣으려면 마우스로 툴바를 클릭해야 합니다. 키보드만으로 빠르게 작성하는 흐름(`Flow`)이 끊깁니다.
- **이미지 배치 제한**: 이미지를 드래그 앤 드롭으로 넣을 수는 있지만, 사이즈 조절이나 레이아웃(좌/우/중앙) 변경이 직관적이지 않습니다.

---

## 2. 개선 목표: "Notion-like Modern Writing Experience"

사용자가 **"글과 콘텐츠에만 집중할 수 있는"** 몰입형 에디터 환경을 구축합니다.

---

## 3. 단계별 실행 계획

### Phase 1: 즉각적인 시각적 개선 (Fix)

**목표**: 커서 문제 해결 및 텍스트 가독성 확보

- [x] `tiptap.css` 수정: `blockquote` 및 기타 요소의 강제 이탤릭(`italic`) 스타일 제거.
- [ ] 커서 가시성 강화: 커서 색상을 브랜드 컬러(Green)로 통일하고 두께를 조정하여 현재 위치를 명확히 표시.
- [ ] Placeholder 개선: 제목과 본문의 Placeholder를 명확히 구분하고, "빈 줄"마다 힌트 제공 고려.

### Phase 2: 작성 효율성 증대 (Enhance)

**목표**: 마우스 없이 키보드만으로 작성 가능한 환경

- [ ] **Slash Command (`/`) 도입**:
  - 본문에서 `/`를 입력하면 팝업 메뉴가 뜨고, 화살표 키로 제목, 목록, 이미지, 인용구 등을 바로 삽입할 수 있는 기능 구현.
- [ ] **Floating Menu 최적화**:
  - 텍스트를 드래그했을 때 나오는 메뉴(Bold, Color 등)가 더 빠르고 정확한 위치에 뜨도록 라이브러리 업데이트.

### Phase 3: 구조적 고도화 (Advanced)

**목표**: 자유로운 레이아웃 및 콘텐츠 배치

- [ ] **Drag & Drop Block Handle**:
  - 각 문단(블록) 왼쪽에 `⋮⋮` 핸들을 표시하여, 마우스로 잡고 위아래로 끌어서 순서를 바꿀 수 있게 구현.
- [ ] **이미지 리사이징 & 정렬 UI**:
  - 이미지 클릭 시 모서리에 핸들이 나와 크기를 조절하고, 직관적인 플로팅 바로 정렬(좌/우/중앙/너비꽉참) 설정 가능하게 변경.
- [ ] **AI Writing Assistant 연동**:
  - 텍스트 선택 시 "AI 요약", "톤 앤 매너 변경", "이어 쓰기" 버튼 제공 (기존 AI 모듈 활용).

---

## 4. 기대 효과

- **집필 속도 2배 향상**: 마우스 이동 없이 키보드(`Slash Command`)만으로 서식 적용 가능.
- **직관적 편집**: 드래그 앤 드롭으로 문단 순서를 자유자재로 변경.
- **프리미엄 사용자 경험**: 깔끔하고 현대적인 UI로 창작자의 만족도 증대.

---

**제안**: 우선 **Phase 1**을 즉시 적용하여 커서 문제를 해결하고, 사용자 피드백을 반영하여 **Phase 2 (Slash Command)** 개발을 착수하는 것을 추천합니다.
